Notable Things Learned:

  • Super() calls code from superclass.
  • Abstract class can't be instantiated.
  • Abstract methods, declared in abstract classes, don't have bodies and child classes actually implement the abstract methods.

The difference between abstract classes and interfaces: You can only extend ONE class but you can implement multiple interfaces. Any field or instance variable declared in an interface is static and final, so values must be assigned and it applies to all classes implementing it. Thus, it makes more sense to put these instance variables in an abstract class.

Abstract class (generalize behavior): use when you have closely related classes that you want to have the same functionality and fields available

Interface (standardize behavior): have unrelated classes that you want to do a certain thing. All methods in an interface must be implemented in the child class that implements the interface.
